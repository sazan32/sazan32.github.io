  • The main goal of the project is to implement an obfuscation tool to prevent static program analysis of applications, in order to thwart (or delay) reverse engineering of software.
    In static program analysis, the code is mostly reviewed by a human, and not executed. The common methods for this are disassembly and decompilation of executable files.
    Reverse engineering of software poses a great risk for organizations of various needs. Some of these are:
    * The loss of intellectual property (the implementation details, etc.)
    * Countering of licensing mechanisms in order to crack proprietary software (by means of patching checks or developing license key generating software)
    * Development of cheats for games (especially online games)
    Static analysis is not the only method for reverse engineering software. Dynamic analysis is another method, which involves execution of the program (or parts of it) in controlled environments, in order to observe how it behaves.

    Dynamic analysis and countermeasures are out of scope for this project.

    Our goal is to develop an obfuscator that transforms the original program into a more complex program, without interfering with the original function.
    This is a challenging task, since our tool will have to analyze non-obfuscated programs, and make a mess while keeping the integrity. From the ordinary user’s perspective, nothing must change. But the potential reverse engineering attempt should be as difficult as can be. Reverse engineering of software can never be completely prevented, since a CPU will have to execute code without errors, and that code will be available for the reverse engineer. But, obfuscation proves itself useful, the skill required for reversing obfuscated code is higher, and it will definitely consume more time and resources, and might render the attempt non-viable.

  • Whilst developing this project,
    * We made use of Zydis - Zyantific Disassembler . It is a disassembly library for C++ and Python.
    * We had to learn about Opaque Predicates, which is also an exciting topic by itself.
    * We still have further improvements on our project, which we plan to implement throughout next months.
    By this point, our program can obfuscate basic C/C++ programs which are compiled with x86 architecture (in 32 bits).
